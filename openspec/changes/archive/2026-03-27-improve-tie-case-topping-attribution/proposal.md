## Problem

平手（tie）情境下，大叔的回應會把聯集配料當成這碗確定有的東西來說——例如 北北車 vs 阿興 平手時，因阿興有香菜，大叔說「那個香菜擺在那裡，真的嘸通啦！」，讓使用者誤以為這碗確定有香菜。

## Root Cause

`_format_input()` 在 tie 時取各候選店 known_toppings 的聯集，但輸出格式是「配料：有 香菜（阿興 的特色）」，仍以「有」開頭，模型讀到後當成事實描述，而非條件敘述。System prompt 雖有加說明，但模型不穩定地遵循。

## Proposed Solution

1. **改變 `_format_input()` 的 tie 配料輸出格式**：分成「兩家都有的配料」與「若是 XX 那碗：可能有 YY（非確定，視店家而定）」兩類，讓模型從資料結構本身理解哪些是條件式。
2. **更新 system prompt**：加入「配料標注說明」，明確指示當配料後面有「若是 XX 那碗」標記時，回應須使用條件語氣（「如果是 XX 的話，可能有...」），不可當成確定事實。

## Success Criteria

- 北北車 vs 阿興 tie 時，回應中若提到香菜，必須用條件語氣（「如果是阿興那碗」或類似說法）
- 兩家都有的配料（如肉鬆同時出現在多家）可直接陳述，不需加條件
- 非 tie 情境（確定店家）的配料描述行為不受影響
