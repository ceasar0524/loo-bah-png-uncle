## Context

大叔的 system prompt 在【語氣規則】和【說話方式】兩個區塊控制回應風格。問題在於原本的規則沒有限制開場白多樣性，也沒有說明天外飛來一筆的使用頻率。

## Goals / Non-Goals

**Goals:**

- 讓大叔的開場白有變化，不會每次都一樣
- 明確天外飛來一筆比喻的使用節制

**Non-Goals:**

- 改變大叔的整體個性或語氣風格
- 修改 few-shot 範例

## Decisions

### 開場白變化規則

在【語氣規則】新增一條：「每次回應的開頭要不一樣，絕對不能每次都用『哎唷』開頭」。直接點名最常見的問題用語，讓模型有明確依據。

### 比喻和描述的節制使用

在【說話方式】新增一條：「比喻和描述以緊扣魯肉飯本身為主，偶爾可以天外飛來一筆，但不能每次都這樣，要有節制。」保留創意空間，同時避免過度使用。

## Risks / Trade-offs

- 模型對「偶爾」的解讀可能仍有偏差，但相比完全禁止，保留彈性更符合大叔個性
