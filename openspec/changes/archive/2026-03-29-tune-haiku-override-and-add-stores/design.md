## Context

Haiku override 機制以特徵加分方式決定是否覆蓋 CLIP 結果。原始閾值 0.5 過低，單一碗色即可觸發，導致碗色相同的店家互相干擾（例如晴光與司機俱樂部同為綠色碗，碗色加分 0.5 已達舊閾值）。

同時，新增 5 家店擴充資料集，並補充視覺特徵定義以提升 Haiku 辨識精度。

## Goals / Non-Goals

**Goals:**

- 提升閾值至 0.75，要求多個特徵同時命中才觸發覆蓋
- 建立同色碗競爭邏輯：晴光需碗色 + 黃蘿蔔（0.8），司機俱樂部僅碗色（0.5）→ 無黃蘿蔔時兩家平手 → 交 CLIP
- 新增 silver/metal 碗支援矮仔財辨識
- 補充 pickled_radish 描述避免背景誤判

**Non-Goals:**

- 不修改 CLIP embedding 或 KNN 演算法
- 不增加或變更 Haiku 模型 prompt 的其他部分

## Decisions

### 閾值從 0.5 提升至 0.75

碗色加分（0.5）單獨不應觸發覆蓋。需要至少碗色 + 一個配料（0.5 + 0.3 = 0.8）或香菜（0.5 + 0.8 = 1.3）才能確信識別。

**Why not lower**: 0.5 會讓任何有 distinctive 碗的店家在碗色匹配時即覆蓋，誤觸機率高。

### 司機俱樂部加入 distinctive 綠碗

讓兩家同色碗進入競爭後，無配料時兩家各得 0.5 形成平手，override 不觸發，CLIP 接手（司機俱樂部 CLIP 準確率 100%）。有黃蘿蔔時晴光得 0.8 勝出，符合預期行為。

**Trade-off**: 若 Haiku 誤判黃蘿蔔（如黃色桌面），晴光仍會蓋掉司機俱樂部。但 eval 結果（晴光 83%、司機俱樂部 42%）已被接受為可接受的平衡點。

## Risks / Trade-offs

- [Risk] Haiku 誤判 pickled_radish（黃色桌面/器皿）→ 司機俱樂部仍會被誤判為晴光（7/12 張）→ 已接受此 trade-off
- [Risk] 新店家照片數量較少，CLIP 準確率可能偏低 → 未來可補充照片改善
